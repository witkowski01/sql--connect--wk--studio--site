Privacy Policy

Last updated: April 13, 2026

Introduction

This Privacy Policy describes how SQL Connect WK Studio ("the App") handles your information. The App is designed to manage SQL databases locally and remotely on your iOS and iPadOS device.

Data Collection

SQL Connect WK Studio does not collect, transmit, or share any personal data with the developer or any third parties. The App does not include analytics, tracking, or advertising frameworks.

Data Storage

The App stores the following connection data locally on your device:

  • Passwords — stored exclusively in the iOS Keychain, protected by hardware-level encryption. With iCloud Sync enabled, passwords synchronize via iCloud Keychain using Apple's end-to-end encryption.
  • Host addresses & usernames — stored locally using standard iOS storage (UserDefaults). With iCloud Sync enabled, they synchronize via Apple's iCloud key-value storage, protected by the user's Apple account.
  • Connection settings (port, SSL, timeout, provider type) — stored alongside host and username using the same mechanisms.
  • SQL scripts — stored locally using on-device storage. With iCloud Sync enabled, they synchronize via Apple's iCloud key-value storage.

iCloud Sync

SQL Connect WK Studio offers an optional iCloud synchronization feature, disabled by default. When enabled by the user in Settings, the following data is synchronized across the user's Apple devices:

  • Connection configurations (name, host address, username, port, SSL settings) — synchronized via Apple's iCloud key-value storage (NSUbiquitousKeyValueStore), protected by the user's Apple account.
  • Connection passwords — synchronized via iCloud Keychain, which uses Apple's end-to-end encryption. Passwords are never stored in plaintext outside of iOS Keychain.
  • SQL scripts — synchronized via Apple's iCloud key-value storage.

All synchronized data is stored exclusively in the user's personal Apple iCloud account. No data is transmitted to any external servers or third parties.

Users can disable iCloud sync at any time in the app's Settings. When disabled, all data remains on the device and no further synchronization occurs. Users can also reset all cloud data from the Settings screen.

Network Communications

The App communicates only with database servers that you explicitly configure. No data is sent to any other servers. When connecting to remote databases, the App uses the connection protocol specified by the user, with optional SSL/TLS encryption.

Local Network Access

The App may request permission to access your local network to connect to database servers running on your local network. This access is used exclusively for database connections that you initiate.

Data Security

The App leverages iOS platform security features including Keychain Services for credential storage and iCloud Keychain for secure cross-device password synchronization. Connection data synced via iCloud is protected by Apple's iCloud security infrastructure. The developer does not have access to your database credentials, connection data, or any data stored in the databases you manage.

Third-Party Services

SQL Connect WK Studio does not integrate with any third-party analytics, advertising, or data collection services.

Frequently Asked Questions

How do I connect to a database?

Tap the "+" button on the Connections screen, select your database type (MSSQL, PostgreSQL, MySQL, or SQLite), enter your connection details, and tap Save. Then tap the connection to connect.

How does iCloud Sync work?

Go to Settings and enable "iCloud Sync". Your saved connections (including passwords via iCloud Keychain) and SQL scripts will automatically sync across all your Apple devices signed in to the same iCloud account. You can disable sync at any time.

Is my data secure?

Yes. Passwords are stored in the iOS Keychain with hardware-level encryption. When iCloud Sync is enabled, passwords sync via iCloud Keychain with Apple's end-to-end encryption. Connection configurations sync via Apple's iCloud key-value storage, protected by your Apple account. The app contains no analytics, tracking, or third-party services.

What database types are supported?

SQL Connect WK Studio supports Microsoft SQL Server (MSSQL), PostgreSQL, MySQL, and SQLite databases.

Can I use the app offline?

Yes. SQLite databases work entirely offline. The SQL Script Catalog is available offline. Remote database connections require network access to the database server.

System Requirements

SQL Connect WK Studio requires iOS 18.0 or later, and is compatible with iPhone and iPad.
